# Insightify 사전 예약 랜딩 페이지

이 문서는 AI 기반 생산성 앱인 Insightify의 사전 예약 랜딩 페이지에 대한 설명입니다.

## 데이터 수집

이 랜딩 페이지는 사용자 행동을 이해하고 제품 및 마케팅 노력을 개선하기 위해 다양한 사용자 상호작용 데이터를 수집합니다. 수집된 모든 데이터는 `user_actions`라는 Supabase 테이블에 저장됩니다.

다음은 수집되는 데이터에 대한 세부 정보입니다.

### 1. 페이지 뷰 이벤트 (`page_view`)
*   **트리거:** 사용자가 페이지에 처음 방문할 때.
*   **수집 데이터:**
    *   `path`: 방문한 페이지의 URL 경로 (예: `/`).
    *   `referrer`: 현재 페이지로 연결된 이전 페이지의 URL.
    *   `user_agent`: 사용자의 브라우저 및 운영 체제 정보.
    *   `screen_width`: 사용자의 화면 너비 (픽셀).
    *   `screen_height`: 사용자의 화면 높이 (픽셀).
    *   `session_id`: 각 사용자 세션에 대해 생성되는 고유 ID.
*   **수집 이유:** 전반적인 웹사이트 트래픽, 사용자 유입 경로, 사용 환경을 이해하기 위함입니다. 이는 사용자 참여도를 분석하고 다양한 기기에 맞게 사이트를 최적화하는 데 도움이 됩니다.

### 2. 스크롤 깊이 이벤트 (`scroll_depth`)
*   **트리거:** 사용자가 페이지의 특정 지점(25%, 50%, 75%, 100%)까지 스크롤할 때.
*   **수집 데이터:**
    *   `depth`: 스크롤한 페이지의 비율.
    *   `session_id`: (위와 동일)
*   **수집 이유:** 사용자가 콘텐츠를 얼마나 소비하는지 이해하기 위함입니다. 스크롤 깊이가 높으면 흥미로운 콘텐츠를 나타내고, 낮으면 개선이 필요한 영역을 시사할 수 있습니다.

### 3. 기능 클릭 이벤트 (`feature_click`)
*   **트리거:** 사용자가 기능 카드 버튼을 클릭할 때.
*   **수집 데이터:**
    *   `feature`: 클릭된 기능의 이름 (예: "자동화된 작업 관리").
    *   `session_id`: (위와 동일)
*   **수집 이유:** 어떤 기능이 사용자에게 가장 흥미로운지 파악하여 개발 및 마케팅 노력을 우선순위화하는 데 도움이 됩니다.

### 4. CTA 클릭 이벤트 (`cta_click`)
*   **트리거:** 사용자가 사전 예약 섹션의 주요 CTA 버튼을 클릭할 때.
*   **수집 데이터:**
    *   `button_text`: 클릭된 CTA 버튼의 텍스트 (예: "얼리버드 혜택받기").
    *   `session_id`: (위와 동일)
*   **수집 이유:** 실제 전환 전에 주요 CTA에 대한 사용자 관심을 측정하기 위함입니다. 이는 CTA의 배치 및 문구를 최적화하는 데 도움이 됩니다.

### 5. 전환 이벤트 (`conversion`)
*   **트리거:** 사용자가 대기 목록 양식을 성공적으로 제출할 때.
*   **수집 데이터:**
    *   `form`: 전환을 유도한 양식의 이름 (예: "waitlist").
    *   `session_id`: (위와 동일)
*   **수집 이유:** 성공적인 가입 또는 목표 달성을 추적하기 위함입니다. 이는 랜딩 페이지의 효과를 평가하는 핵심 지표입니다.

## 데이터 수집 확인

데이터가 수집되는지 확인하려면 다음을 수행할 수 있습니다.

1.  **브라우저 개발자 콘솔 확인:** 콘솔을 열고(일반적으로 F12 또는 Cmd+Option+I) 성공적인 이벤트 로깅을 나타내는 `console.log` 메시지(예: "Event logged successfully:")를 찾습니다.
2.  **네트워크 요청 모니터링:** 개발자 도구에서 "네트워크" 탭으로 이동하여 "XHR" 또는 "Fetch"로 필터링합니다. 이벤트가 트리거될 때 Supabase 프로젝트 URL로 요청이 전송되는 것을 볼 수 있습니다.
3.  **Supabase 프로젝트 로그 확인:** Supabase 대시보드에 로그인하여 프로젝트로 이동한 다음, `user_actions` 테이블의 "로그" 섹션을 확인합니다.
