# 🚀 가상 신제품 랜딩페이지 사용자 행동 수집 시스템

## 1. 프로젝트 개요

- **프로젝트명:** "Insightify" - AI 기반 생산성 앱 사전예약 페이지 사용자 행동 분석 시스템
- **목표:** 마케팅 포트폴리오 활용을 위해, 사용자의 행동 데이터를 수집하고 분석하는 시스템을 구축한다. 이를 통해 데이터 기반의 의사결정 및 마케팅 퍼널 최적화 역량을 증명한다.
- **핵심 컨셉:** 사용자가 랜딩페이지에 들어와서 사전예약(전환)을 완료하기까지의 고객 여정(Customer Journey)을 추적하고, 각 단계별 전환율을 측정할 수 있는 데이터 포인트를 수집한다.

## 2. 기술 스택

- **Frontend:** HTML, CSS, Vanilla JavaScript (Vite)
- **Backend & DB:** Supabase
- **Deployment:** (추후 결정, 예: Netlify, Vercel)

## 3. 수집할 사용자 행동 데이터 (Data Points)

| 데이터 항목 (Event) | 수집할 정보 | 수집 이유 (마케팅적 관점) | 분석 지표 |
| --- | --- | --- | --- |
| **1. 페이지 조회 (Page View)** | 접속 시간, 접속 국가, 사용 기기 | 어떤 사용자가, 언제, 어떤 환경에서 우리 페이지에 관심을 보이는지 파악 (든지' 단계) | 총 방문자 수, 재방문율, 기기별/국가별 분포 |
| **2. 스크롤 깊이 (Scroll Depth)** | 도달한 스크롤 위치 (25%, 50%, 75%, 100%) | 사용자가 페이지의 어느 부분까지 흥미를 느끼고 이탈하는지 파악 (관심' 단계) | 스크롤 단계별 이탈률 |
| **3. 핵심 기능 클릭 (Feature Click)** | 클릭한 기능명 | 잠재 고객이 가장 매력적으로 느끼는 제품의 핵심 기능(USP) 파악 | 기능별 클릭률 |
| **4. 사전예약 버튼 클릭 (CTA Click)** | 버튼 클릭 이벤트 | 전환(사전예약)에 대한 사용자의 '의도'를 측정 (고려' 단계) | CTA 클릭률 (방문자 대비) |
| **5. 사전예약 완료 (Conversion)** | 고유 식별자, 예약 완료 시간 | 최종 목표(KPI) 달성 여부를 측정 (전환' 단계) | 최종 전환율 (CTA 클릭 대비 / 방문자 대비) |

## 4. 주요 기능 요구사항

- **FR1:** 사용자가 랜딩페이지에 접속하면 `Page View` 이벤트가 Supabase에 기록되어야 한다.
- **FR2:** 사용자가 페이지를 스크롤할 때, 특정 지점(25%, 50%, 75%, 100%)을 통과하면 `Scroll Depth` 이벤트가 Supabase에 기록되어야 한다. (단, 각 지점은 한 번만 기록)
- **FR3:** 사용자가 핵심 기능 소개 섹션의 버튼을 클릭하면 `Feature Click` 이벤트가 Supabase에 기록되어야 한다.
- **FR4:** 사용자가 '사전예약하기' 버튼을 클릭하면 `CTA Click` 이벤트가 Supabase에 기록되어야 한다.
- **FR5:** 사용자가 이메일을 입력하고 '제출' 버튼을 눌러 유효성 검사를 통과하면 `Conversion` 이벤트가 Supabase에 기록되어야 한다.
